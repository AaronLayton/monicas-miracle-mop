## Directory Purposes

**app/**
- Purpose: Next.js App Router - defines pages, routes, and root layout
- Contains: Server/Client components, page definitions, global layout, styles
- Key files: `page.tsx` (home route), `layout.tsx` (root layout), `globals.css` (theming)

**components/**
- Purpose: Reusable React components
- Contains: UI components, interactive elements
- Key files: `ui/button.tsx` (shadcn/ui Button wrapper)

**components/ui/**
- Purpose: UI component library - styled primitives
- Contains: shadcn/ui components, Base UI wrapped components
- Key files: `button.tsx` (multi-variant button component)

**lib/**
- Purpose: Utility functions and helpers
- Contains: Helper functions, shared logic
- Key files: `utils.ts` (className composition utility)

**public/**
- Purpose: Static assets served at `/` route
- Contains: Images, icons, fonts (if self-hosted)
- Key files: SVG logos (next.svg, vercel.svg), example assets

**docs/**
- Purpose: Design documentation and brand guidelines
- Contains: Design mockups, brand guidelines, marketing materials
- Key files: Design iterations (design-[1-4]-*), brand guidelines, logo assets

**.planning/codebase/**
- Purpose: Auto-generated codebase documentation
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, STACK.md, INTEGRATIONS.md, CONCERNS.md
- Generated by: GSD codebase mapper tools

**.vercel/**
- Purpose: Vercel deployment configuration
- Contains: Deployment settings, project configuration
- Generated by: Vercel CLI or dashboard

## Key File Locations

**Entry Points:**
- `app/layout.tsx`: Root layout - loaded for every page, sets up HTML, fonts, global styles, metadata
- `app/page.tsx`: Home page route (/)

**Configuration:**
- `next.config.ts`: Next.js config (typed routes enabled, cache components enabled, dev indicators position)
- `tsconfig.json`: TypeScript compiler options (strict mode, path aliases, ES2017 target)
- `components.json`: shadcn/ui config (style: base-nova, RSC enabled, Tailwind CSS, path aliases)
- `eslint.config.mjs`: ESLint config (flat config format, Next.js rules + Core Web Vitals)
- `postcss.config.mjs`: PostCSS config (Tailwind CSS v4 plugin)

**Core Logic:**
- `lib/utils.ts`: Utility function for className merging (cn utility)
- `components/ui/button.tsx`: Button component implementation with CVA variants

**Styling:**
- `app/globals.css`: Global styles, CSS custom properties (design tokens), dark mode variables, Tailwind theme
- Theme colors: primary, secondary, accent, destructive, muted, background, foreground, border, etc.
- Uses OKLch color format for CSS custom properties

## Naming Conventions

**Files:**
- `.tsx`: React Server Components (default in app/) and Client Components
- `.ts`: Utility files, helpers, types
- `.mjs`: ESM modules for config files (eslint, postcss)
- PascalCase for component files: `Button.tsx`, `Page.tsx`
- camelCase for utility/helper files: `utils.ts`

**Directories:**
- lowercase for feature/domain directories: `app/`, `components/`, `lib/`, `public/`
- kebab-case for nested directories with multiple words: `components/ui/`
- Descriptor prefixes for docs: `design-[1-4]-*` for design iterations

**Components:**
- Exported as default from component files in components/ui/
- Named exports for subcomponents (e.g., `export { Button, buttonVariants }`)

**CSS Classes:**
- Tailwind utility classes with arbitrary values: `bg-zinc-50`, `dark:bg-black`
- CSS custom properties via Tailwind theme: `var(--background)`, `var(--font-geist-sans)`
- Selector-based dark mode: `.dark` class on html element

## Where to Add New Code

**New Page/Route:**
- Create `app/[route-name]/page.tsx` for new routes
- Use `app/layout.tsx` pattern for nested layouts
- Pages are Server Components by default
- Add `"use client"` at top of file for client-side interactivity

**New UI Component:**
- Add to `components/ui/` directory
- Use Base UI primitives as unstyled base
- Apply styling via CVA (class-variance-authority) for variants
- Use `cn()` utility for className composition
- Export both component and variants (e.g., `buttonVariants`)

**New Feature/Business Logic:**
- Create domain directory under `components/` (e.g., `components/dashboard/`, `components/checkout/`)
- Group related components by feature
- Can include server or client components as needed

**Shared Utilities:**
- Add to `lib/` directory
- Export as named exports
- Follow naming: `lib/[domain].ts` (e.g., `lib/auth.ts`, `lib/formatting.ts`)

**Static Assets:**
- Add SVGs, images, fonts to `public/` directory
- Reference via `/[filename]` (e.g., `src="/next.svg"`)
- Supports images via Next.js `Image` component or HTML `img`

**.planning/codebase/:**

- Committed: Yes (for reference by CI/CD and team)
