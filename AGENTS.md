# Monica's Miracle Mop - Project Guide

## Overview

Professional website for Monica's Miracle Mop, Kasey's domestic house cleaning business based in the UK. Built with Next.js 16 App Router, Tailwind CSS v4, and shadcn/ui (base-nova style). The site enables customers to browse services, schedule appointments, and book cleaning jobs. Booking confirmation events are added to Google Calendar and confirmation emails sent via Resend.

## Tech Stack

| Layer | Choice | Notes |

## Design System

### Colour Palette (from globals.css :root — all oklch)

The base-nova defaults are neutral. Phase 1 design conversion applies the design-3 purple/teal palette. The CSS variable names to use throughout:

| Token | Usage |
|-------|-------|
| `--primary` | Headings, primary actions (purple) |
| `--primary-foreground` | Text on primary backgrounds |
| `--secondary` | Secondary actions, accents (teal) |
| `--secondary-foreground` | Text on secondary backgrounds |
| `--accent` | Light teal highlights |
| `--accent-foreground` | Text on accent backgrounds |
| `--background` | Page background |
| `--foreground` | Body text |
| `--muted` | Subtle backgrounds |
| `--muted-foreground` | Subdued text |
| `--border` | Borders and dividers |
| `--destructive` | Error states |

Use semantic colour classes only — `bg-primary`, `text-secondary`, `border-border`. Never use hardcoded hex values or arbitrary Tailwind colours like `bg-purple-500`.

### Glass-Card Effects (globals.css utilities)

Three utility classes for glassmorphism effects matching the design-3 aesthetic:
- `.glass-card` — `backdrop-blur(16px)`, 70% white background — general use
- `.glass-card-nav` — `backdrop-blur(20px)`, 85% white background — navigation bar
- `.glass-card-sidebar` — `backdrop-blur(12px)`, 70% white background — booking summary sidebar

### Component Conventions

- UI primitives live in `components/ui/` (generated by shadcn CLI — do not hand-edit)
- Domain components live in feature directories: `components/booking/`, `components/schedule/`
- Shared layout components at `components/` root: `nav.tsx`, `footer.tsx`
- All components use `data-slot="component-name"` attribute (see button.tsx for reference)
- Use `cn()` from `@/lib/utils` for className merging
- Use CVA (`class-variance-authority`) for component variants
- Use composable sub-components: `CardHeader` / `CardTitle` / `CardContent` / `CardFooter`
- Import primitives from `@base-ui/react/[component]` — NOT `@radix-ui/react-[component]`
- Use `gap-*` for flex/grid spacing, NOT `space-x-*` or `space-y-*`

### Button Pattern (reference implementation)

```tsx
"use client"
import { Button as ButtonPrimitive } from "@base-ui/react/button"
import { cva, type VariantProps } from "class-variance-authority"
import { cn } from "@/lib/utils"

function Button({ className, variant, size, ...props }) {
  return (
    <ButtonPrimitive
      data-slot="button"
      className={cn(buttonVariants({ variant, size, className }))}
      {...props}
    />
  )
}
```

### Server/Client Component Boundary

- Pages (`app/*/page.tsx`) are Server Components — no `"use client"` at page level
- Interactive leaf components use `"use client"`: `CalendarPicker`, `CounterInput`, `MobileNavToggle`
- Shared layout components (`Nav`, `Footer`, `BookingSummary`) are Server Components in Phase 1
- Only add `"use client"` when the component needs `useState`, `useEffect`, or event handlers

## UK Localisation Rules

- All prices in GBP (£) — no USD amounts anywhere
- UK English spelling in all user-visible text: colour, organised, centre, favourite
- UK address format: Address Line 1, Address Line 2, Town/City, County, Postcode
- UK phone format: 07700 900000
- `lang="en-GB"` on the `<html>` element
- No US-specific terminology: use "flat" not "apartment", "postcode" not "zip code"
- Copyright and legal text must reference UK law where applicable
